This README provides documentation for the R scripts used to process positioning data and generate visualisations for the Žužić et al. manuscript. These scripts handle the transformation of raw `.pos` files into statistical summaries, high-quality PDF plots, and statistical testing.

***

# Positioning Error Analysis and Visualisation

This repository contains two primary R scripts designed to analyse Global Navigation Satellite System (GNSS) positioning errors during ionospheric storm events (Storms 1–4).

## 1. Scripts Overview

### `make_time_series.R`
**Purpose:** This is the primary data processing and temporal analysis script.
- **Data Parsing:** It scans directories for `.pos` files, extracts GPST (GPS Time) data, and aligns it with geographic coordinates.
- **Distance Calculation:** It calculates geodesic distances (using the `geosphere` library) between raw data points and a fixed reference coordinate (`ref_coords`) to determine:
    - Horizontal positioning errors (meters).
    - Absolute latitude/longitude deviations.
    - Vertical (height) errors.
- **Statistical Output:** - Generates summary statistics (Min, Q1, Median, Mean, Q3, Max, SD) grouped by hour.
    - Formats and prints LaTeX-ready tables for academic publication.
- **Visualisations:** Produces multi-day time series plots showing the fluctuation of horizontal errors across the duration of each storm event.
- **Output Files:** Saves processed data for each individual storm/day as a CSV file.

### `make_circular_plot.R`
**Purpose:** Generates advanced circular (polar) distribution plots for the manuscript.
- **Proportional Analysis:** It reads the CSV files generated by the time-series script and bins positioning errors into categorical ranges.
- **Circular Visualisation:** Uses `ggplot2` with `coord_polar` to display error distributions by hour of the day.
    - Features a "hollow center" design with a specialised "gap" at the 12 o'clock position for percentage scales.
    - The Viridis colour pallete is colourblind-safe.
- **Formatting:** Applies scientific notation (`plotmath`) to legend labels and ensures all labels are tangentially rotated for readability.
- **Output Files:** Saves high-resolution PDF files.

### `make_stat_test.R`
**Purpose:** Generates results for inferential circular statistical tests for the manuscript.
- **Helper Function to Extract Circular Peak Errors:** This function isolates the top 25% of errors (Q3 and above) to represent the peak error times.
- **Rayleigh Test for Uniformity:** Tests if the errors are randomly distributed across the 24 hours (Null Hypothesis) or if they have a significant directional (temporal) cluster (Alternative Hypothesis, p < 0.05 indicates clustering).
- **Watson-Williams Tests (Circular ANOVA):** Compares the mean angular directions (peak error times) between different storms.
- **Mardia-Watson-Wheeler Tests (Non-Parametric):** Evaluates whether two or more circular distributions are identical and it is robust against low concentration parameters and non-von Mises distributions.
- **Output Files:** No files, the output is generate in the terminal.

## 2. Requirements

### R Packages
Ensure you have the following libraries installed:
```r
install.packages(c("circular", "tidyverse", "dplyr", "tidyr", "lubridate", "geosphere", "ggplot2", "readr", "stringr", "extrafont"))
```

### System Dependencies
- **Fonts:** The scripts use "DejaVu Sans". You may need to install this font on your system and run `extrafont::font_import()`.
- **PDF Rendering:** The scripts use `cairo_pdf` for high-quality vector output, requiring a working Cairo installation on your OS.

---

## 3. Data Structure

The scripts expect the following directory structure:
```text
.
├── make_time_series.R
├── make_circular_plot.R
├── make_stat_test.R
├── Storm 1/
│   └── (Multiple .pos files)
├── Storm 2/
│   └── (Multiple .pos files)
├── Storm 3/
│   └── (Multiple .pos files)
└── Storm 4/
│   └── (Multiple .pos files)
```

---

## 4. Usage

1.  **Process Data:** Run `make_time_series.R` first. This will crawl the "Storm" folders, calculate errors, generate the necessary `.csv` files, and produce the time-series PDF plots.
2.  **Generate Circular Plots:** Run `make_circular_plot.R`. It will look for the `.csv` files produced in the previous step and generate the polar distribution plots.
3. **Conduct Statistical Tests** Run `make_stat_test.R`. You can observe the test statistics and $p$-values in the terminal output.

---
